# Software Designer Glossary

[1] WRSPM - World, Requirements, Specification, Program, Machine. Reference model for requirements analysis.

[2] VT (Visi&on) - Ontology roles: agent (does), object (acted upon), transfer (moves), transaction (exchange).

[3] Abduction - Inference to the best explanation from partial evidence.

[4] KADS - Knowledge Acquisition and Documentation Structuring. Expertise modeling methodology.

[5] Means-Ends - Problem-solving technique to reduce gap between current and desired state.

[6] CSP - Constraint Satisfaction Problem. Variables with domains + constraints that must be satisfied.

[7] Natural Deduction - Formal inference rules: ∧ (and), ∨ (or), → (implies), ¬ (not).

[8] Tableaux - Proof by systematic decomposition or contradiction.

[9] Hoare Logic - {Pre} program {Post}. Contract-based verification.

[10] CBR - Case-Based Reasoning. Adapt solutions from similar past problems.

[11] AHP - Analytic Hierarchy Process. Weighted criteria decision-making.

[12] TRIZ - Russian acronym. 40 inventive principles for resolving technical contradictions.

[13] Six Thinking Hats - Parallel thinking: White(facts), Red(emotion), Black(caution), Yellow(benefits), Green(creativity), Blue(control).

[14] PMI - Plus, Minus, Interesting. Evaluation framework.

[15] ADR - Architecture Decision Record. Lightweight decision documentation.

[16] Type-Driven Development - Design with types to make illegal states unrepresentable.

[17] Sum Type - Tagged union. Mutually exclusive options.

[18] Product Type - Record/tuple. Composite data.

[19] Invariant - Property that must hold across all states.

[20] Result/Either - Type for explicit error handling.
